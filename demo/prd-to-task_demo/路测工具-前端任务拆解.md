# 【路测工具】前端开发任务拆解清单

> 基于 PRD：`【PRD】路测工具.pdf`
> 拆解视角：前端开发（FE）
> 拆解原则：原子级、单一职责、可估时（0.5~1d）、可验收

---

## 一、全局上下文摘要

- **核心业务价值**：为 MiniVan 6 Plus 无人车量产下线提供路测/疲劳测试的数字化管控平台，实现质量状态可视、问题闭环、数据可追溯。
- **前端覆盖范围**：监控系统单车页"创建路测任务"、综合/疲劳测评浮窗、运管系统路测测评管理、路测测试项管理、路测地图设置、路测方案管理、缺陷提报配置、机器人通知 H5 卡片。
- **不覆盖**：车端安卓屏发起路测（已暂缓）、压线检测前端、后端算法、3D 整车实时回放底层渲染（仅复用既有组件）。
- **用户角色**：京东质检员（O 岗）、长安质检员。
- **核心主流程**：单车页创建路测任务 → 立即出发 → 实时测评（任务中）→ 已结束查看回放并测评 → 提交/作废 → 已测评/已作废归档。

## 二、领域拆解（前端视角）

```
Epic: 路测工具前端
├── Feature A：创建路测任务（监控-单车页右侧浮窗）
├── Feature B：实时/回放测评浮窗（综合 & 疲劳）
├── Feature C：路测测评管理列表（综合/疲劳页签）
├── Feature D：路测测试项管理
├── Feature E：路测地图设置
├── Feature F：路测方案管理
├── Feature G：缺陷提报配置
└── Feature H：机器人通知页面（H5 静态卡片）
```

2. **画龙检测合格标准**：行驶距离阈值"先按 5 米需支持配置"——是否需要前端提供配置入口？建议改为后端配置。
3. **地图图层刻度尺（FE-E03）**：巩方汇调研中，地图组件能否原生输出图层长宽？若不能，前端需自行像素↔米换算（+0.5d 工时风险）。
4. **跑行趟次下发车端**：刘智需调度系统支持，前端字段已规划（FE-F05），需明确 `supportLap` / `lapCount` 字段契约。
5. **3D 整车回放视频切片**：FE-B10/B11 假设复用既有"整车回放"组件，需确认能否按"停靠点-停靠点"切片传参。
6. **缺陷提报路径**：列表字段含"提报路径"但弹窗无对应编辑入口，需澄清前端是否可配置。
7. **机器人推送页面归属**：PRD 显示是"卡片原型"，建议确认是 H5 链接页（前端做）还是机器人服务端模板（后端做）。
8. **暂停=急停？**：FE-C04 中"暂停"按钮是否下发急停？需与监控系统现有急停按钮交互对齐。
9. **画龙/坡起/碰撞等自动化检测项**：测试方法枚举与 PRD 6.4/6.5 表格中的检测条件保持一致即可，前端无需校验阈值。
10. **多人并发编辑配置**：PRD 未要求乐观锁，按"后写覆盖"实现，需用户确认。
